# Prompt Engineering Skill

Converts raw user intent into production-ready, structured prompts. Works across domains and AI
agents. Optimized for clarity, token efficiency, and deterministic output.

---

## Workflow

When this skill triggers, follow these steps in order:

1. **Extract intent** — Identify what the user actually wants the AI to DO (not just say)
2. **Select role** — Choose the most effective persona for the target model
3. **Expand context** — Add the minimum necessary background to eliminate ambiguity
4. **Generate constraints** — Define what the output must and must not include
5. **Format output spec** — Specify structure, length, tone, and format
6. **Compress** — Remove redundant tokens while preserving all signal
7. **Return structured prompt** — Deliver the final prompt + a debug note if the original was weak

---

## Core Prompt Framework

Every prompt generated by this skill follows this structure:

```
[ROLE]       — Who the AI is playing (expertise, perspective, voice)
[TASK]       — The single, precise action to perform
[CONTEXT]    — Background the AI needs; no more, no less
[CONSTRAINTS]— Explicit rules: length, tone, format, forbidden content
[OUTPUT]     — Exact specification of what the response must look like
```

**Adaptive rules:**
- Omit `[ROLE]` only for purely factual single-turn queries
- Expand `[CONTEXT]` when domain knowledge is required
- Use numbered `[CONSTRAINTS]` when there are 3 or more rules
- Always include `[OUTPUT]` — never leave format implicit

## Output Contract

Every response from this skill must include:

1. **Structured Prompt** — the final, ready-to-use prompt in a code block
2. **Prompt Score** — 1-line assessment of the original input (if one was provided)
3. **Key Changes** — bullet list of what was added/removed/restructured
4. **Usage Note** — which AI agents this prompt is optimized for

## Quick Reference: Prompt Patterns

```
Simple task:      [ROLE] + [TASK] + [OUTPUT]
Complex task:     [ROLE] + [TASK] + [CONTEXT] + [CONSTRAINTS] + [OUTPUT]
Chain step:       [PRIOR OUTPUT] + [ROLE] + [TASK] + [HANDOFF FORMAT]
System prompt:    [IDENTITY] + [CAPABILITIES] + [BOUNDARIES] + [VOICE]
Debug mode:       [ORIGINAL] + [FAILURE MODE] + [FIXED VERSION] + [REASON]
```
